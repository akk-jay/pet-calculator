# Google AdSense 人工审核通过指南（通用版）

> 适用：工具站、计算器站、博客、内容站等各类网站的 AdSense 首次申请。
> 基于 2024-2026 年大量真实案例（通过与被拒）的系统性总结。

---

## 一、理解审核：Google 审核员到底在看什么

### 1.1 核心认知

AdSense 是一个**上下文广告系统**。它的商业模式是：

```
爬虫读懂你的内容 → 匹配相关广告 → 广告主付费 → Google 分成给你
```

如果 Google **读不懂**你的页面内容，就无法匹配广告，就不会让你通过。

因此，审核的核心标准只有一个：**你的页面是否提供了足够的、可被爬虫理解的高质量原创内容。**

### 1.2 审核流程（两个阶段）

| 阶段 | 做什么 | 耗时 |
|------|--------|:--:|
| **自动机审** | 爬虫扫描全站 → 检测内容量、原创度、技术合规 | 即时~几天 |
| **人工审核** | 审核员打开网站 → 逐页浏览 → 主观判断"这个站有没有价值" | 1-14 天 |

两个阶段都必须通过。很多人倒在自动机审（内容量不够），更多人倒在人工审核（第一眼就觉得"没价值"）。

### 1.3 审核员的 10 秒第一印象

人工审核员打开你的网站，**前 10 秒**决定了他对网站的基本判断：

| 他看到什么 | 他的判断 |
|-----------|---------|
| 干净的排版、清晰的导航、首屏有大量文字 | ✅ "看起来是个正经网站" |
| 首屏只有一个输入框和一个按钮 | ❌ "工具壳，没有内容" |
| 到处都是广告位（即使空的） | ❌ "广告位比内容多，拒" |
| 文字是机翻/乱码/堆砌关键词 | ❌ "垃圾站" |
| AI 味很重的内容（空洞、重复） | ❌ "AI 批量生成，拒" |
| 404 页面、死链、加载缓慢 | ❌ "不专业" |

## 二、硬性门槛清单（必须 100% 达标）

以下条件**缺一不可**，任何一条不满足都会被直接拒绝：

### 2.1 基础设施

| 序号 | 要求 | 说明 | 代价（不满足） |
|:--:|------|------|------|
| 1 | **自定义顶级域名** | `.com` / `.net` / `.org` 等。禁止免费子域名（`github.io`、`vercel.app`、`blogspot.com`） | 直接拒绝 |
| 2 | **全站 HTTPS** | 证书有效，无混合内容警告 | 直接拒绝 |
| 3 | **移动端适配** | 375px 宽度下不横向滚动、按钮可点击、文字可读 | 高概率拒绝 |
| 4 | **页面加载速度** | Core Web Vitals 全绿（LCP < 2.5s, INP < 200ms, CLS < 0.1） | 中概率拒绝 |
| 5 | **所有页面可访问** | 无 404、无死链、无"建设中"页面 | 高概率拒绝 |

### 2.2 必备页面

| 序号 | 页面 | 要求 |
|:--:|------|------|
| 1 | **隐私政策页（Privacy Policy）** | 独立页面。声明：(1) Cookie 用途 (2) 广告网络收集的数据类型 (3) 第三方服务（Google Analytics 等）(4) 用户权利。**页脚链接可见。** |
| 2 | **关于页面（About）** | 你是谁、为什么做这个网站、你的专业背景。150-300 字即可。**必须有真人感。** |
| 3 | **联系我们（Contact）** | 真实的联系表单或邮箱地址。不能只是一个 mailto 链接。建议用表单。 |
| 4 | **服务条款（Terms of Use）** | 强烈建议有。简单声明网站内容用途、免责声明即可。 |

> ⚠️ 以上页面必须在**网站首页 1 次点击可达**（通常放页脚）。

### 2.3 内容量要求

| 最低标准 | 推荐标准 | 说明 |
|:--:|:--:|------|
| **15 个页面** | **25-30 个页面** | 这是"内容页面"的数量，不包括 About/Contact/Privacy 等合规页面 |
| **每页 ≥800 字** | **每页 1200-2000 字** | 原创文字，不能是 AI 直出、不能是复制粘贴 |
| **每个页面有独立价值** | — | 不能是同一模板换了几个词的"变体页面" |

### 2.4 时间要求

| 要求 | 说明 |
|------|------|
| **域名年龄 ≥2-3 个月** | 新注册域名（<1 个月）立刻申请，几乎必被拒 |
| **内容发布时间跨度 ≥1 个月** | 不要让审核员看到 25 篇文章全部是同一天发布的 |
| **两次申请间隔 ≥3-4 周** | 被拒后不要立刻重新提交。必须先实质性改进 |

## 三、页面内容标准（决定能否通过的核心）

### 3.1 什么是"高质量内容"

Google 判断内容质量的 E-E-A-T 框架：

| 维度 | 含义 | 如何体现 |
|------|------|---------|
| **Experience（经验）** | 你有第一手经验 | 写你自己的真实经历、案例、数据 |
| **Expertise（专业）** | 你懂这个领域 | 正确的术语、深入的解释、引用权威来源 |
| **Authoritativeness（权威）** | 你是该领域的可靠来源 | 被其他网站引用、有外链、作者有背景介绍 |
| **Trustworthiness（信任）** | 网站是可信的 | 透明（谁做的、怎么联系）、准确（无错误信息）、安全（HTTPS） |

### 3.2 内容自查清单（逐页检查）

```
☐ 这个页面回答了一个明确的问题吗？
   用户搜 "装修预算怎么算" → 你的页面给了完整答案 = ✅
   用户搜 "装修预算怎么算" → 你的页面只有一个计算器 = ❌

☐ 这个页面即使是纯文本也有价值吗？（把计算器去掉，剩下的文字有没有用？）
   有用 = ✅ 设了计算器反而更弱 = ❌

☐ 页面有清晰的层级结构吗？（H1 → H2 → H3，不是全用 div）
☐ 段落长度合适吗？（每段 2-4 句，无人阅读的长文块 = 没有价值）
☐ 有原创的数据、表格、对比、案例吗？（不是泛泛而谈）
☐ 有没有引用来源并给出链接？
☐ 文字读起来像真人在说话吗？（不是 ChatGPT 味）
☐ 拼写和语法正确吗？
```

### 3.3 AI 内容警示

Google **不禁止** AI 辅助创作，但**检测并惩罚**纯 AI 生成的低质量内容。

| AI 辅助（可以通过） | AI 直出（会挂） |
|------|------|
| AI 生成草稿 → 人工大改 → 加入个人经验 | AI 生成原文 → 直接发布 |
| 有具体案例、数据、个人观点 | 泛泛而谈、空洞、重复 |
| "我们去年给 50 个客户做装修预算，发现…" | "装修预算是一个重要的环节。做好预算可以帮助你省钱。" |
| 段落长短不一，节奏自然 | 每个段落都是完美的 3 句话，结构格式化 |

**快速检测：** 把你的文字读出来。如果听不出"人味"，审核员也看不出来。

## 四、不同类型网站的通过策略

### 4.1 工具站 / 计算器站（最高风险）

**核心问题：** Google 认为"工具不是内容"。功能再好，爬虫只看到 `<input>` 和 `<button>`。

**通过策略：** "内容包裹工具"模式

```
每个工具页面的结构：

┌─────────────────────────────────────┐
│ H1 标题（含目标关键词）                │
│ 导语：200-300 字                      │  ← 告诉 Google 这个页面讲什么
├─────────────────────────────────────┤
│ ★ 工具交互区 ★                       │  ← 首屏可见
├─────────────────────────────────────┤
│ H2：如何使用这个工具（200-300 字）     │
│ 1-2-3 步骤 + 截图                    │
├─────────────────────────────────────┤
│ H2：相关知识点（500-800 字）★ 最核心  │  ← 这才是审核员看到的内容
│   H3 × 2-3 个子话题                  │
│   数据、对比、技巧、案例               │
├─────────────────────────────────────┤
│ H2：常见问题 FAQ（4-6 个问答）        │
│ 使用 FAQPage 结构化数据标记 ★         │
├─────────────────────────────────────┤
│ 相关页面推荐（4-6 个内链）            │
└─────────────────────────────────────┘

文字总量：每页 1200-2000 字
```

**额外要做的事：**

- 开一个 Blog 栏目，每个工具对应 1-2 篇深度文章
- Blog 文章和工具页互相链接
- 首页要有"内容导览"区域（最新文章、推荐工具），不要只放工具列表
- **工具页占全站比例 < 60%**。如果全站 20 个页面，18 个是工具，仍然危险。

### 4.2 内容站 / 博客

**核心问题：** 内容质量和原创性。

**通过策略：**

| 要求 | 说明 |
|------|------|
| 25+ 篇文章 | 少于 15 篇基本过不了 |
| 每篇 1000+ 字 | 少于 500 字的文章不算"实质内容" |
| 聚焦一个主题 | 不要今天写装修、明天写养猫、后天写编程。Google 需要判断你的"专业领域" |
| 文章间的时间跨度 | 至少有 4-6 周的发布历史 |
| 有作者信息 | 每篇文章标注作者 + 简短介绍 |

### 4.3 产品展示站 / 落地页

**核心问题：** 如果网站目的是推广你自己的产品/服务，AdSense 不会批准。

**通过策略：** 这种类型的网站**不建议申请 AdSense**。建议用 Google Ads 投放广告，而不是接广告。

### 4.4 社区 / UGC 网站

**核心问题：** Google 需要看到原创内容，用户生成内容（论坛帖子、评论）不被视为"你的内容"。

**通过策略：**

- 你必须自己产出大量原创内容（≥60% 的页面是你写的）
- 有严格的内容审核机制
- 删除低质量/灌水 UGC

## 六、申请流程与时间线

### 6.1 推荐时间线

```
第 1-2 月：建站期
  ├── 第 1 周：搭框架、设计页面模板
  ├── 第 2-4 周：产出 10-15 篇核心内容
  ├── 第 5-6 周：产出 10-15 篇补充内容
  └── 第 7-8 周：完善合规页面、检查技术细节

第 3 月：观察期
  ├── 确保 Google 已收录（site:你的域名）
  ├── 修复 Search Console 里的问题
  ├── 补充被忽略的长尾内容
  └── 月末：第一次提交 AdSense 申请

第 4 月：结果
  ├── 通过 → 添加广告代码 → 开始变现
  └── 拒绝 → 精读拒绝原因 → 针对性补内容 → 等 3-4 周再申请
```

### 6.2 申请步骤

| 步骤 | 操作 |
|:--:|------|
| 1 | 访问 [adsense.google.com](https://adsense.google.com) |
| 2 | 填写网站 URL（`https://你的域名`，不要带 `www.` 除非你的规范域名就是 www） |
| 3 | 填写收款国家/地区信息 |
| 4 | 勾选协议 |
| 5 | **不要在页面上放置 AdSense 代码**（审核通过后会给你） |
| 6 | 等待 1-14 天 |

### 6.3 提交后不要做的事

```
❌ 不要大幅修改网站结构（审核员打开时发现和提交时不一样）
❌ 不要添加/删除页面
❌ 不要改动域名解析
❌ 不要让网站宕机
❌ 不要每天刷新 AdSense 后台看结果
```

## 七、常见拒绝原因及修复方案

### 7.1 拒绝原因速查表

| 拒绝原因（英文） | 中文含义 | 根本问题 | 修复方案 |
|------|------|------|------|
| **Low value content** | 低价值内容 | 页面内容太少，或只有工具没有文字 | 每页补充 1000+ 字原创内容 |
| **Insufficient content** | 内容不足 | 总页面数不够（<15 个） | 增加到 25+ 个内容页面 |
| **Screens without publisher content** | 无发布者内容的页面 | 工具/交互页面没有文字内容 | 在工具页面加入文章区 |
| **Site is down or unavailable** | 网站无法访问 | 审核时网站挂了 | 确保 uptime |
| **Site does not comply with policies** | 违反政策 | 有违规内容（版权问题、成人内容等） | 逐条检查 AdSense 政策 |
| **Under construction** | 网站建设中 | 有空页面、Lorem Ipsum、占位符 | 删除所有未完成页面 |
| **Difficulty navigating** | 导航困难 | 菜单混乱、找不到关键页面 | 优化导航结构 |
| **Scraped / duplicate content** | 采集/重复内容 | 内容不是原创 | 重写所有页面 |

### 7.2 被拒后的正确操作

```
被拒
  ↓
精读拒绝原因（只有一句话，但每个词都重要）
  ↓
不要立刻重新提交 ← 关键！！！
  ↓
对比第七章的修复方案，逐项整改
  ↓
整改完成后，让网站自然运行 3-4 周
  ↓
确认新内容已被 Google 收录
  ↓
再次提交申请
```

**被拒 2 次以上：** 暂停申请，等待 2-3 个月，大幅改版内容策略（不是小修小补），再提交。

## 八、特殊情况

### 8.1 GitHub Pages 子域名能过吗？

```
github.io 子域名 → 通过率极低（<10%）
自定义域名 + GitHub Pages → 可以（HTTPS 自带 ✅）
```

**建议：** 先买一个域名（¥50-100/年），解析到 GitHub Pages，用自定义域名提交申请。

### 8.2 中文网站能过吗？

**可以。** AdSense 支持中文网站，审核员也有中文母语的。但注意：

- 内容质量要求和其他语言一样
- 中文 RPM 通常比英文低（$1-5 vs $8-25）
- 如果做中英双语版，建议先集中做一种语言申请

### 8.3 网站有 AI 生成内容能过吗？

| 做法 | 结果 |
|------|------|
| AI 生成 → 直接发布，无人工干预 | ❌ 极高概率被拒 |
| AI 生成 → 人工修改 30-50% → 加入个人经验/数据 | ⚠️ 看运气 |
| AI 生成 → 人工重写 70%+ → 补充原创案例/数据 | ✅ 可以通过 |
| 完全人工写 | ✅ 最优 |

### 8.4 网站还没有流量，能申请吗？

**可以申请，而且这是正常情况。** AdSense 不需要你有流量才申请，它审核的是"内容质量"而不是"流量大小"。但在审核期间至少要有一些自然访问记录（Search Console 里有数据），因为零流量新站看起来像"还没被用户验证过"。

## 九、备选方案（如果 AdSense 持续不通过）

| 广告平台 | 门槛 | 收益 | 适合 |
|------|:--:|------|------|

| **联盟营销（Amazon/京东联盟）** | 无审核门槛 | 佣金 1-10% | 内容站/评测站 |
