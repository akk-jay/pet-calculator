# 纯前端工具站标准化上线流程

> 适用于：纯前端单页工具站（图片转换、PDF 处理、格式压缩等），零后端、零服务器成本，依靠广告变现。

---

## 阶段一：基础设施搭建

### 1.1 GitHub 仓库 + Pages 部署

| 步骤 | 操作 | 耗时 |
|------|------|:--:|
| `git init` | 项目目录初始化 | 1 分钟 |
| `gh repo create` | 创建公开仓库并推送 | 2 分钟 |
| GitHub Pages 启用 | `gh api .../pages -X POST` 或网页手动开启 | 2 分钟 |

**产出：** `xxx.github.io/repo-name/` 免费子域名，立即可访问。

**坑点：** 确保仓库 `source` 分支选对（`master` 或 `main`），路径设为 `/`。

### 1.2 自定义域名 + HTTPS

| 步骤 | 平台 | 说明 |
|------|------|------|
| 挑选域名 | 阿里云 / Cloudflare | `.net` > `.cc` > `.com`，避开 `.cn`（免备案） |
| 购买 + 实名 | 阿里云 | 约 50-100 元/年 |
| DNS A 记录 x4 | 阿里云 DNS 解析 | 记录值：`185.199.108.153` / `109.153` / `110.153` / `111.153`，主机记录 `@` |
| DNS CNAME | 阿里云 DNS 解析 | 主机记录 `www`，记录值 `xxx.github.io` |
| CNAME 文件 | GitHub 仓库根目录 | 内容为域名（如 `imgconv.net`） |
| SSL 等待 | — | GitHub 自动签发 Let's Encrypt，5 分钟~几小时 |
| 强制 HTTPS | GitHub Pages 设置页 | 证书签发后手动勾选 |

**产出：** `https://你的域名` 全球可访问，HTTPS 全绿。

**优化：** 购买和 DNS 都在一个平台做最省事。Cloudflare 买域名自动配 DNS，阿里云买域名需要手动加记录。

## 阶段二：产品开发

### 2.1 架构约束

- **单 HTML 文件**：内联 CSS + JS，部署零构建步骤
- **纯前端处理**：Canvas API / Web Crypto / WASM，不涉及任何后端
- **CDN 渐进加载**：重型库（WASM、JSZip）首次使用时动态加载，首页不阻塞

### 2.2 必备模块（按优先级排列）

| 模块 | 说明 | 必须？ |
|------|------|:--:|
| 隐私提示条 | 顶部固定栏：「所有处理均在本地浏览器完成，不上传服务器」 | ✅ |
| 核心功能区 | 上传→处理→下载，页面最显眼位置 | ✅ |
| 操作指引 | 拖拽/点击上传 + 一句话说明 | ✅ |
| 成果预览 | 缩略图 + 前后大小对比 | ✅ |
| 扩展功能卡片 | 可展开/折叠，默认折叠保持页面清爽 | 视功能多少 |
| SEO 内容区 | 200-300 字说明 + H2/H3 + FAQ 结构化数据 | ✅ |
| 隐私政策页 | 独立页面，声明本地处理、Cookie 用途、第三方服务 | ✅（AdSense 刚需） |
| Cookie 横幅 | 底部固定：「Cookie 仅用于广告和统计」，一键关闭 | ✅ |
| 广告占位 | 侧边栏（桌面）+ 底部横幅 + 移动插屏 x2，`<div>` 占位 | ✅ |

### 2.3 视觉规范

- **配色**：单一品牌色渐变（Tailwind CSS 变量体系，HSL 值便于暗色模式扩展）
- **卡片**：白色 + 圆角 12-16px + 柔和阴影
- **按钮**：品牌色渐变主按钮，绿色下载按钮，灰色次要按钮
- **格式/选项标签**：药丸形（圆角 9999px），选中态品牌色填充，未选中浅灰边框
- **响应式**：1024px 双栏（主内容+侧边广告），768px 单栏，480px 紧凑

### 2.4 性能边界

- 单文件上限 10MB，超限 Toast 提示
- 转换/处理完立即释放临时内存（`revokeObjectURL`、Canvas 置空）
- 5MB 以内单文件处理 ≤200ms，批量 10 个 ≤2s

## 阶段三：搜索引擎 + 分析

### 3.1 Google Search Console

| 步骤 | 操作 |
|------|------|
| 添加资源 | 选 **网址前缀** 方式，填入 `https://你的域名` |
| 验证方式 | 推荐 **DNS TXT 记录**（阿里云加一条 TXT，最快最稳） |
| 备用验证 | 同时在页面 `<head>` 里加 meta 标签，防止文件误删导致验证丢失 |
| 提交站点地图 | 左侧菜单→站点地图→填入 `sitemap.xml` |

**产出：** Google 1-3 天内开始收录。

### 3.2 Bing Webmaster

| 步骤 | 操作 |
|------|------|
| 添加网站 | 选 **「从 Google Search Console 导入」**，一键完成，零额外验证 |

### 3.3 流量统计

- 推荐 **Cloudflare Web Analytics**：免费、不设 Cookie、一行 JS
- 注册 → Add site → 填入域名 → 复制 token → 放到页面 `</body>` 前

### 3.4 SEO 配套文件

| 文件 | 内容 |
|------|------|
| `sitemap.xml` | 单页 URL + `lastmod` + `changefreq` + `priority` |
| `robots.txt` | 指向 sitemap 位置 |
| 结构化数据 | WebApplication schema + FAQPage schema（JSON-LD 格式） |
| Meta 标签 | Title、Description、Keywords、OG、Canonical |

## 阶段四：变现上线

### 4.1 Google AdSense 申请

| 步骤 | 说明 |
|------|------|
| 前提条件 | 域名已上线、HTTPS 正常、隐私政策页存在 |
| 提交申请 | adsense.google.com → 填域名 → 勾选协议 |
| 放置代码 | 把 `<script async src="...adsbygoogle.js?client=ca-pub-xxx">` 放到 `<head>` |
| 创建 ads.txt | 根目录放 `ads.txt`，内容 `google.com, pub-xxx, DIRECT, f08c47fec0942fa0` |
| 开启自动广告 | AdSense 后台 → 网站 → 开启自动广告 |
| 等待审核 | 1-4 周，期间广告可能不展示或少量展示 |

**坑点：**
- 不要在自动化浏览器（Claude/Playwright）里申请 AdSense，会被判非真人拒绝
- GitHub Pages 子域名（`xxx.github.io`）审核通过率低，先买域名再申请
- CMP 设置选 Google 自带「2 选项（同意+管理）」，最简单合规

### 4.2 通过审核后的收尾

- 把 4 个广告占位 `<div>` 替换为 AdSense 自动广告代码或手动广告单元
- 确认自动广告已启用，等 1-2 小时开始展示

### 4.3 收款配置

- AdSense 后台 → 付款 → 付款信息 → 添加银行卡
- 门槛：收入满 **$100** 打款
- 与网站代码无关，随时可填

## 快速参考：关键命令

```bash
# 创建仓库 + 部署
gh repo create <name> --public --push
gh api repos/<user>/<repo>/pages -X POST -F "source[branch]=master" -F "source[path]=/"

# DNS 验证（TXT 记录）
nslookup -type=TXT <domain> 8.8.8.8

# HTTPS 证书检查
echo | openssl s_client -servername <domain> -connect <domain>:443 2>/dev/null | openssl x509 -noout -subject

# 验证代码是否部署
curl -s https://<domain> | grep <keyword>
```
