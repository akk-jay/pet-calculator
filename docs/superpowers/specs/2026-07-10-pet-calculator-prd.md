# PetToolKit 宠小算 — 产品需求文档 (PRD)

> 版本：v1.1 | 日期：2026-07-11 | 状态：已评审 | 作者：产品团队

---

## 1. 文档概述

### 1.1 文档目的

本文档定义 **PetToolKit 宠小算** 产品的业务目标、目标用户、功能需求与非功能需求。供产品、设计、开发、运营相关方对齐预期。

技术实现细节参见 `2026-07-10-pet-calculator-design.md`。

### 1.2 产品一句话

一个面向养宠人群的在线工具套件网站，提供年龄换算、喂食计算、预产期推算、驱虫周期、疫苗时间表 5 个实用计算器，通过 AdSense 广告变现。

## 2. 背景与机会

### 2.1 市场现状

| 发现 | 来源 |
|------|------|
| 通用计算器巨头（Omni Calculator 3000万月访、Calculator.net 5200万月访）覆盖宠物工具品类，但仅是 3700+ 工具中的 5-8 个，不构成宠物专属体验 | 竞品调研 |
| 市面现有 ~22 个宠物计算器小站，但均为单一功能（只做年龄或只做成本），**没有一个一站式的宠物计算器平台** | Web 搜索调研 |
| GitHub 上仅 ~11 个宠物计算器仓库，最高 4 星，多数已废弃，无成熟开源竞品 | GitHub 调研 |
| 中文市场仅 2 个微信小程序（觅惠工具箱、笑哈哈计算器），**无独立网站** | 中文市场调研 |
| 宠物经济持续增长，中国宠物市场规模 2025 年已超 3000 亿元 | 行业公开数据 |

### 2.2 机会判断

**"宠物计算器品类 = 有需求、无品牌"**

- 搜索量存在："dog age calculator" 月搜索量约 30 万（全球），"宠物年龄计算器" 月搜索量约 5000+（中文）
- 没有品类主导者：用户搜到的是 Omni 的宠物子页面或某个单一功能小站
- 如果做到 "宠物计算器首选站 = PetToolKit"，就有品类红利

### 2.3 为什么是我们

| 优势 | 说明 |
|------|------|
| **首发中文市场** | 中文宠物专属计算器独立站目前是空白 |
| **宠物档案系统** | 创建一次宠物信息，所有工具自动预填，降低重复输入摩擦 |
| **零运营成本** | 纯前端 + GitHub Pages，年成本仅域名 ¥50-100 |
| **隐私友好** | 全本地计算，数据不离开用户设备，差异化于数据收集型竞品 |
| **移动端优先设计** | 90% 用户用手机访问，UI 触控尺寸、广告克制策略均从手机端反推 |

## 4. 目标用户

### 4.1 核心用户画像

#### 画像 1：新手宠主 — 小陈

> 25 岁，女，深圳，互联网运营，刚养了一只 3 个月的金毛犬"豆包"

| 场景 | 痛点 | 使用 PetToolKit 的方式 |
|------|------|------|

| 不知道该喂多少 | 狗粮包装袋上的克数范围太大(150-300g) | 用喂食量计算，得到精确克数 |
| 不知道啥时候打疫苗 | 繁育人说了一堆记不住 | 建宠物档案，自动生成疫苗时间表 |

#### 画像 2：准繁育人 — 李先生

> 32 岁，男，成都，自由职业，家中有一只 2 岁母英短准备配种

| 场景 | 痛点 | 使用 PetToolKit 的方式 |
|------|------|------|
| 想算预产期 | 只知道"猫怀孕大概两个月"，不确定 | 用预产期推算，得到精确日期和关键时间线 |
| 准备产房 | 不知道什么时候该准备 | 时间线显示第 58 天准备产房，提前做好规划 |

#### 画像 3：科学养宠派 — 王女士

> 28 岁，女，上海，数据分析师，养有 2 猫 1 狗

| 场景 | 痛点 | 使用 PetToolKit 的方式 |
|------|------|------|
| 管理多只宠物数据 | 每只年龄、体重、疫苗、驱虫日期都不同，记不住 | 创建 3 个宠物档案，一键切换，各自独立计算 |
| 驱虫日程混乱 | 常常忘了哪只什么时候驱过虫 | 驱虫周期计算器显示每只宠物的下次驱虫倒计时 |

## 5. 产品概述

### 5.1 产品结构

```
PetToolKit 宠小算
│
├── 首页                  工具集入口 + 宠物档案管理
│
├── 人宠年龄换算           狗/猫实际年龄 → 人类等效年龄
├── 喂食量计算器           体重+体况+活动量 → 每日喂食克数
├── 宠物预产期推算          配种日期 → 预产期+孕期关键时间线
├── 驱虫周期计算器          生活环境+上次驱虫日 → 未来12个月驱虫日历
├── 疫苗时间表查询          出生日期+地区 → 疫苗时间表+下次接种提醒
│
├── 养宠知识（Blog）       5 篇科普文章，与工具互相导流
│
└── 合规页面               关于/联系/隐私政策/服务条款
```

### 5.2 核心差异化：宠物档案系统

**为什么需要：** 每次使用不同工具都要重新输入品种、体重、年龄，用户会觉得麻烦。

**解决方案：** 首次使用创建一个"宠物档案"（名字 + 猫/狗 + 品种 + 生日 + 体重 + 绝育状态），之后进入任意工具，已填数据自动带入，用户只需补充 1-2 个该工具特有的字段。

**数据存储：** 浏览器本地存储（localStorage），不上传服务器，与产品隐私定位一致。

**交互类比：** 类似 ChatGPT 左侧对话列表，用户可创建多只宠物档案，点击切换。

**展示范围：** 宠物列表**仅在首页显示**（桌面端左侧栏，移动端顶部可收起区域）。工具页不展示侧栏，保持计算器界面专注。用户进入工具页时自动读取当前活跃宠物的数据，无需在工具页切换宠物。

**语言切换：** 页脚放置文字链接"中文 | English"，用户手动选择。不做浏览器自动检测。

### 5.3 内容写作策略

**Blog 文章生产流程：**
1. AI 辅助生成初稿（利用兽医指南、学术论文等参考资料）
2. 人工修改 **30-50%**，加入个人经验、真实案例、原创数据对比
3. 审核 AI 味指标：段落长短不一、有具体数字案例、读出来像真人在说话
4. 最终质检：对照《AdSense 审核通过指南》3.3 节 AI 内容警示标准

> 参考依据：Google 不禁止 AI 辅助创作，但惩罚纯 AI 直出的低质量内容。

**免责声明规范：** 每篇 Blog 文章末尾添加：
> "本文内容仅供参考，不构成兽医建议。如您的宠物有健康问题，请咨询执业兽医。"

### 5.4 平台策略

| 阶段 | 语言 | 目标市场 | 国际化方式 |
|------|------|------|------|
| v1.0 首发 | 纯中文（zh-CN） | 中文用户 | 不做内置 i18n。依赖浏览器自带翻译（Chrome/Safari/Edge）覆盖外语用户。`<html lang="zh-CN">` 声明语言，确保翻译引擎正确识别。 |
| v1.1 | 视流量来源决定 | — | 若英文流量占比 >30%，添加页脚语言切换。否则继续纯中文 + 浏览器翻译。 |

## 6. 功能需求

### 6.1 宠物档案系统

#### F-PET-001：创建宠物档案

| 字段 | 优先级 |
|------|------|
| 宠物名字 | P0 必填 |
| 猫/狗 | P0 必选 |
| 品种 | P1 选填（自由文本） |
| 出生日期 | P1 选填（精确到月） |
| 体重(kg) | P1 选填 |
| 是否绝育 | P1 选填（默认否） |

- **规则：** 必填项不填时，"保存"按钮置灰不可点
- **反馈：** 保存成功后弹窗关闭，宠物列表自动更新，新宠物自动选中

#### F-PET-002：切换宠物

- 左侧宠物列表点击宠物名/头像 → 切换为当前活跃宠物
- 当前活跃宠物高亮显示（暖奶咖底色）
- 进入任意工具页时，读取当前活跃宠物的数据自动预填

#### F-PET-003：编辑/删除宠物

- 宠物条目右侧 hover 时显示 `⋯` 图标按钮，点击弹出"编辑 / 删除"菜单
- 移动端：`⋯` 图标始终可见（不依赖 hover）
- 删除需二次确认："确定要删除 [宠物名] 的档案吗？此操作不可撤销。"
- 编辑弹窗复用创建弹窗，预填已有数据，保存后更新

#### F-PET-004：无宠物引导

- 用户未创建任何宠物时，点击工具卡片 → 弹出引导：
  > "创建宠物档案后，工具将自动预填年龄、体重等信息，省去每次手动输入。"
  - [立即创建] → 弹出添加宠物弹窗
  - [跳过，手动填写] → 直接进入工具页

#### F-PET-005：localStorage 不可用处理

- 当浏览器禁用 localStorage（隐私模式/无痕模式/安全设置）时：
  - 页面顶部显示黄色提示条："当前浏览器不支持本地存储，宠物档案将无法保存。不影响计算器正常使用。"
  - 提示条可关闭（关闭后本次会话不再显示）
  - 所有工具页正常可用，但打开工具时不再弹出"请先创建宠物"引导，直接进入手动填写模式
  - 添加宠物按钮在首页显示但点击时弹出说明"当前浏览器不支持本地存储，无法创建宠物档案"

### 6.2 工具 1：人宠年龄换算

**F-AGE-001：核心技术**

| 宠物类型 | 公式 |
|------|------|
| 狗（通用） | 人类年龄 = 16 × ln(实际年龄) + 31 |
| 猫 | 第 1 年 = 15 人类年；第 2 年 = +9；之后每年 +4 |

- 狗的体型修正：大型犬(>25kg) 老化加速约 10-15%，在结果展示中说明
- 数据来源：狗公式来自 UC San Diego 2020 年发表于 *Cell Systems* 的研究；猫公式来自 AAHA/AAFP 行业指南

**F-AGE-001：输入**

| 输入项 | 来源 | 可手动改 |
|------|------|:--:|
| 宠物类型（狗/猫） | 宠物档案 | ✅ |
| 实际年龄（年 + 月） | 宠物档案出生日期自动计算 | ✅ 可手动滑块调整 |
| 体型（仅狗显示：小型<10kg / 中型10-25kg / 大型>25kg） | — | ✅ 必选 |

**F-AGE-001：输出**

| 输出项 | 展示方式 |
|------|------|
| 人类等效年龄 | 大数字突出，例：**约 47 岁** |
| 生命阶段标签 | 幼年期/青少年期/壮年期/中年期/老年期，配阶段图标 |
| 阶段养护建议 | 1-2 句话，如"你的狗狗正处于精力最充沛的阶段" |
| 换算详情（可折叠） | 展示所用公式、传统 7 倍法对比、算法来源引用 |

**F-AGE-002：异常状态**

| 异常 | 处理 |
|------|------|
| 年龄输入极端值（>30 年） | 显示"请确认年龄是否正确，通常犬猫寿命不超过 30 年"，仍允许计算 |
| 年龄为 0（<1 个月） | 显示"太小啦！建议至少满 1 个月再使用此工具" |
| 未选择体型（狗） | 体型选择区红色边框提示"请选择体型以获得更精确结果"，阻止计算结果展示 |

### 6.3 工具 2：喂食量计算器

**F-FEED-001：核心公式**

```
RER = 70 × 体重^0.75
MER = RER × 生命阶段因子 × 活动水平系数
每日喂食克数 = MER ÷ 食物热量密度 × 100
每日饮水量 ≈ 体重(kg) × 50 ml（粗略建议）
```

**F-FEED-001：输入**

| 输入项 | 来源 | 可手动改 |
|------|------|:--:|
| 宠物类型（狗/猫） | 档案 | ✅ |
| 体重(kg) | 档案 | ✅ |
| 体况评分 BCS（1-9 量表） | — | ✅ 必选，默认 5（标准）。v1.0 使用纯文字描述（"肋骨可摸到但不可见 = BCS 5"），v1.1 补简笔画图示 |
| 活动水平（低/中/高） | — | ✅ 必选，默认中 |
| 是否绝育 | 档案 | ✅ 影响 MER 因子 |
| 生命阶段 | 档案出生日期自动判定 | ✅ 覆盖：幼年/成年/老年/孕期/哺乳 |
| 食物热量密度(kcal/100g) | — | ✅ 默认干粮 380，可自定义 |

**F-FEED-001：输出**

| 输出项 | 展示方式 |
|------|------|
| 每日所需热量(kcal) | 大数字 |
| 每日喂食干粮克数 | 克 + 餐数建议 |
| 每日喂食湿粮克数 | 克 + 餐数建议 |
| 每日建议饮水量 | ml |
| 计算详情（可折叠） | RER → MER → 克数全链路展示 |

**F-FEED-001：MER 因子规则**

| 状态 | 狗 | 猫 |
|------|:--:|:--:|
| 绝育成年 | 1.6 | 1.3 |
| 未绝育成年 | 1.8 | 1.5 |
| 不活跃/易肥胖 | 1.2 | 1.0 |
| 减重期 | 1.0 | 0.8 |
| 幼年 (<4 月) | 3.0 | 2.5 |
| 幼年 (4-12 月) | 2.0 | 2.0 |
| 孕期 (后 1/3) | 2.0 | 1.8 |
| 哺乳期 | 4.5 | 4.0 |
| 老年 (>7 岁) | 1.4 | 1.2 |

活动水平系数：低 0.8 / 中 1.0 / 高 1.2
BCS 偏离标准时，每偏离 1 分调整 MER ±10%

数据来源：AAHA 2021 营养与体重管理指南、Pet Nutrition Alliance、APOP。

**F-FEED-002：异常状态**

| 异常 | 处理 |
|------|------|
| 体重未填（档案无数据 + 用户未手动填） | 输入框高亮橙色边框 + 提示"请填写体重"，结果区显示占位符"--" |
| 体重输入极端值（<0.1kg 或 >200kg） | 显示"请确认体重是否正确"，仍允许计算但附加⚠️标记 |
| 食物热量密度为 0 或负数 | 输入框红色边框，提示"热量密度必须大于 0"，阻止计算结果 |
| BCS 为两端的极端值（1 或 9） | 结果区附加提醒"当前体况评分偏低/偏高，建议咨询兽医制定饮食计划" |

### 6.4 工具 3：宠物预产期推算

**F-PREG-001：核心数据**

| 物种 | 孕期 | 范围 |
|------|:--:|------|
| 狗 | 63 天 | 58-68 天 |
| 猫 | 65 天 | 63-67 天 |

**F-PREG-001：输入**

| 输入项 | 来源 |
|------|------|
| 宠物类型（狗/猫） | 档案 |
| 配种日期 | 日期选择器，用户手动输入（必填） |

**F-PREG-001：输出**

| 输出项 | 展示方式 |
|------|------|
| 预产日期 | 日历视图 + 日期文字 |
| 预产窗口 | ±2 天范围 |
| 当前孕期天数 / 总天数 | 进度条 + 百分比 |
| 倒计时 | "距预产期还有 X 天" |
| 关键节点时间线 | 5 个里程碑节点（B超/X光/准备产房/预产），已完成✅ / 待完成⏳ |
| 产前准备清单 | 纯文本列表（产箱、保暖、毛巾、兽医联系方式等） |

**F-PREG-002：异常状态**

| 异常 | 处理 |
|------|------|
| 未选择配种日期 | 输入框高亮提示"请选择配种日期"，结果区和时间线均不展示 |
| 配种日期为未来日期 | 提示"配种日期不能是未来日期" |
| 配种日期距今超过 70 天（已过预产窗口） | 提示"预产期已过或非常接近，请注意观察母宠状态。如需确认是否已生产，请咨询兽医。" |
| 当前孕期 < 0（日期已过预产窗口 >5 天） | 显示"预产期已过 X 天。如果母宠尚未生产，请立即联系兽医。" |

### 6.5 工具 4：驱虫周期计算器

**F-DEWORM-001：驱虫频率规则**

| 条件 | 体内驱虫 | 体外驱虫 |
|------|:--:|:--:|

### 6.7 通用异常状态（全局）

以下异常状态适用于所有工具页和首页。

#### 6.7.1 localStorage 不可用

| 场景 | 处理 |
|------|------|
| 浏览器禁用 localStorage / 隐私模式 | 页面顶部显示黄色提示条（可关闭）："当前浏览器不支持本地存储，宠物档案将无法保存。不影响计算器正常使用。" |
| 提示条关闭后 | 本次会话不再显示，不影响任何功能使用 |
| 首页添加宠物按钮 | 仍可点击，但弹出说明"当前浏览器不支持本地存储，无法创建宠物档案" |
| 工具页 | 直接进入手动填写模式，不弹出"请先创建宠物"引导 |

#### 6.7.2 极端值输入

所有工具的数值输入框统一规则：
- 检测到极端但可能合理的值（如体重 0.5kg 或 80kg）→ 黄色提示，仍允许计算
- 检测到明显错误的值（如年龄 -5、体重 9999）→ 红色边框，阻止计算
- 空值（必填项未填）→ 高亮提示，结果区显示"--"

#### 6.7.3 直接访问工具页 URL

用户可能通过搜索引擎或分享链接直接进入工具页，此时可能：
- 无宠物档案 → 不弹引导，静默进入手动填写模式
- 有宠物档案 → 自动读取当前活跃宠物数据预填
- URL 应可正常分享，页面状态独立（不依赖从首页跳转的路径）

#### 6.7.4 JavaScript 禁用

- 页面 `<noscript>` 标签显示"本网站需要 JavaScript 才能使用计算器功能，请启用 JavaScript 后刷新。"
- 不追求 JS 禁用下的降级体验（计算器本身依赖 JS）

### 6.8 广告位策略

参照《AdSense 审核通过指南》，广告位分两阶段处理：

**设计原则（移动端优先）：** 90% 用户通过手机访问，广告必须克制。

| 原则 | 规则 |
|------|------|
| 广告位数量上限 | 工具页 ≤2 个，Blog ≤2 个，首页 ≤1 个 |
| 广告位位置 | 仅在"自然阅读断点"——计算结果下方 + 页面底部。不在输入区旁边（防误触） |
| 禁止项 | ❌ 弹窗插屏 ❌ 悬浮固定广告 ❌ 视频自动播放 ❌ 占屏幕 50%+ 的大广告 |
| 广告尺寸（移动端） | 320×100 或 300×250，不占满屏 |
| AdSense 自动广告 | 审核通过后开启，但设为**"低密度"**，手动审核每个自动广告位置 |
| 广告与内容间距 | 广告区域上下各留 ≥24px 间距，与内容卡片明确分隔 |

#### Phase 1：审核前（v1.0 当前状态）

- **不在任何页面放置广告代码**
- 仅在布局上预留广告占位 `div`，样式为浅灰虚线边框 + 居中文字"广告位"
- 占位位置（移动端优先，桌面端兼容）：
  - 工具页：计算结果下方（横幅 320×100）+ 页面底部 FAQ 之前（矩形 300×250）
  - 首页：工具卡片下方（横幅）
  - Blog：文章中部段落间（矩形）1 个
- 占位 div 无网络请求，不影响加载速度
- 占位 div 以注释 `/* AD_PLACEHOLDER */` 标记，方便批量替换

#### Phase 2：审核通过后（v1.1+）

- 替换占位 div 为 AdSense 代码
- 启用 AdSense 自动广告 → 设为低密度 → 手动审核位置
- 严格按移动端优先规则控制广告密度

#### 关键约束

| 约束 | 说明 |
|------|------|
| 审核期间零广告 | 提交 AdSense 申请前及审核期间，页面完全干净，不含任何广告代码 |
| 广告不能压过内容 | 审核员打开手机版网站 → 首屏必须看到工具和文字，不能先看到广告 |
| 不遮挡交互 | 广告不能紧贴输入区/按钮，避免误触和体验恶化 |
| 不自动弹窗 | 不使用自动弹出的插屏广告（违反 AdSense 政策且 Google 会惩罚移动端排名） |
| 不闪烁/CLS | 广告必须预留固定高度容器，加载前后不引起页面布局跳动 |

## 7. 非功能需求

### 7.1 性能

| 指标 | 要求 |
|------|------|
| 页面首次加载（LCP） | < 2 秒 |
| 交互响应（INP） | < 100 毫秒 |
| 累计布局偏移（CLS） | < 0.05 |
| 单页总资源大小 | < 200 KB |

### 7.2 兼容性（移动端优先）

**90% 用户通过手机访问 → 移动端体验优先于桌面端。**

| 平台 | 优先级 | 要求 |
|------|:--:|------|
| 手机 | **P0** | iOS Safari（iPhone 主力）、微信内置浏览器（Android 常用）、Android Chrome。375px 宽度无横向滚动，触控目标 ≥44×44px |
| 平板 | P1 | iPad Safari, Android 平板 Chrome |
| 桌面 | P2 | Chrome/Safari/Firefox/Edge 近 2 年版本 |

**移动端必测机型：**
- iPhone SE（375px）→ 最小宽度，不能有任何横向滚动
- iPhone 14/15（390px）→ 主流 iOS
- Android 中端机（360-412px）→ 微信内置浏览器

### 7.3 安全与隐私

| 要求 | 说明 |
|------|------|
| 全站 HTTPS | 强制 |
| 用户数据不上传 | 所有宠物档案数据仅在浏览器 localStorage 中 |
| 无用户追踪 | 不使用第三方 Cookie（Google Fonts 除外） |
| 隐私政策可访问 | 页脚链接，1 次点击可达 |

### 7.4 无障碍

| 要求 | 优先级 |
|------|:--:|
| 所有输入框关联 label | P0 |
| 色彩对比度 ≥ AA 级（文字 4.5:1，大字 3:1） | P1 |
| 键盘可操作所有功能 | P1 |
| 图片有 alt 文本 | P2 |

### 7.5 SEO

| 要求 | 说明 |
|------|------|
| 每个页面独立 `<title>` + `<meta description>` | 含目标关键词 |
| 结构化数据 | 工具页：WebApplication + FAQPage (JSON-LD)；文章页：Article |
| Open Graph 标签 | 所有页面可被社交分享 |
| `<link rel="canonical">` | 避免重复内容惩罚 |
| sitemap.xml | 提交至 Google Search Console + Bing Webmaster |
| robots.txt | 允许全部收录，指向 sitemap |

### 7.6 国际化 (i18n)

**v1.0 策略：纯中文 + 浏览器翻译，不做内置切换。**

| 决策 | 说明 |
|------|------|
| 页面语言 | `<html lang="zh-CN">`，全部中文 |
| 外语用户覆盖 | 依赖浏览器自带翻译（Chrome/Safari/Edge），零开发成本 |
| 翻译引擎兼容 | 语义化 HTML、文字不用图片、JS 动态文字用 `textContent` |
| 不做的事 | ❌ 不建语言切换按钮、❌ 不写英文版内容、❌ 不做 IP/浏览器语言检测 |
| v1.1 触发条件 | 若 Search Console 数据显示英文搜索流量占比 >30%，再考虑添加页脚语言切换 |

## 9. 风险与缓解

| 风险 | 概率 | 影响 | 缓解措施 |
|------|:--:|:--:|------|
| AdSense 审核被拒 | 中 | 高 | 参考《AdSense 审核通过指南》；每页 1200+ 字原创内容；域名先运行 2-4 月再申请；被拒后等 4 周再提交 |
| 搜索引擎不收录 | 低 | 高 | sitemap + Search Console + 结构化数据 + 内链策略；参考《部署流程文档》 |
| 工具公式被质疑不准确 | 中 | 中 | 每页注明科学来源（论文/指南），展示计算详情，加免责声明"仅供参考，详细请咨询兽医" |
| 流量增长缓慢 | 中 | 中 | 首发做长尾低竞争关键词；养宠社区（豆瓣/小红书/Reddit）内容分发；6 个月达不到 3000 UV 则启动内容营销 |
| 浏览器 API 变更导致计算器异常 | 低 | 低 | 每 6 个月功能回归测试；无第三方依赖，风险可控 |
