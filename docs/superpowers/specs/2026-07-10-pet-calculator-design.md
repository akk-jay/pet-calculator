## 1. 产品概述

### 1.1 产品定位

**PetToolKit 宠小算** 是一站式宠物实用工具套件网站，为养宠人群提供在线计算工具，帮助宠主科学养宠、精准测算猫狗日常养护数据。

### 1.2 核心价值主张

| 维度 | 描述 |
|------|------|
| **谁在用** | 中国养狗/养猫人群，90% 通过手机搜索具体养宠问题进入 |
| **解决什么** | 养宠过程中的"不知道怎么算"——年龄换算、喂食量、预产期、驱虫/疫苗时间 |
| **为什么选我们** | 移动端优先设计、宠物档案一键预填、科学公式透明展示、隐私安全（数据不离开设备） |
| **赚钱方式** | Google AdSense 广告 + 宠物用品联盟营销 |
| **语言策略** | v1.0 纯中文，依赖浏览器自带翻译覆盖外语用户（`<html lang="zh-CN">` + 语义化 HTML） |

### 1.3 商业模式

```
免费工具 → 搜索引擎流量 → AdSense 广告展示 → 广告收入
                          → 联盟营销链接（Amazon/Chewy）→ 佣金收入
```

- **年运营成本：** ¥50-100（仅域名续费）
- **收支平衡所需流量：** 月 UV ~3,000-5,000
- **稳定盈利所需流量：** 月 UV ~30,000

### 1.4 竞品差异化

| 对比维度 | 通用计算器站（Omni/Calculator.net） | 品牌计算器（Purina/皇家） | 宠物小站 | **PetToolKit** |
|----------|:--:|:--:|:--:|:--:|
| 宠物专属 | ❌ 只是3700个工具中的几个 | ⚠️ 仅限自家产品 | ✅ | ✅ |
| 工具齐全度 | ⭐⭐ | ⭐ | ⭐⭐ | ⭐⭐⭐⭐ |
| 中文支持 | ❌ | ❌ | ⚠️ 极少 | ✅ 中英双语 |
| 宠物档案预填 | ❌ | ❌ | ❌ | ✅ localStorage 一键带入 |
| 科学公式透明 | ⭐⭐⭐ | ⭐ | ⭐ | ⭐⭐⭐ |
| 隐私安全 | ⚠️ 不确定 | ❌ 收集数据 | ⚠️ | ✅ 全本地计算 |

**核心护城河：** 宠物档案系统 + 中英双语 + 工具齐全度，三者组合目前市场空白。

### 4.5 响应式断点（移动端优先）

**设计顺序：先做完 375px 移动端，再向上适配桌面端。** 90% 用户手机访问。

| 断点 | 宽度 | 布局策略 | 优先级 |
|------|------|------|:--:|
| Mobile S | ≥375px | **基准设计宽度**。单栏，宠物列表折叠到顶部可展开区，工具卡片1列，触控目标 ≥44×44px | **P0** |
| Mobile L | ≥414px | 同上，间距略微放宽 | P0 |
| Tablet | ≥768px | 工具卡片2列，宠物列表仍折叠 | P1 |
| Desktop | ≥1024px | 左侧宠物列表(220px)常驻 + 主内容区 + 工具卡片4列 | P2 |

**移动端特定约束：**
- 375px 宽度下无任何横向滚动
- 所有可点击元素最小触控区域 44×44px（iOS HIG 标准）
- 输入框字号 ≥16px（防止 iOS 自动缩放）
- 广告容器预留固定高度，防止加载时 CLS

### 4.6 浏览器翻译兼容

v1.0 纯中文，依赖浏览器翻译覆盖外语用户。技术保障：

```html
<html lang="zh-CN">  <!-- 关键：声明语言让翻译引擎正确识别 -->
```

```css
/* 避免翻译引擎漏掉动态文字 */
.result-text { }
/* 所有 JS 渲染的文字使用 textContent 而非 innerHTML */
```

```javascript
// ✅ 翻译引擎能识别
el.textContent = '你的狗狗相当于人类 47 岁';

// ❌ 翻译引擎可能漏掉
el.innerHTML = '你的狗狗相当于人类 <b>47</b> 岁';  // 如需加粗，用 textContent + CSS
```

- `<h1>`、`<p>`、`<li>` 语义化标签确保翻译引擎按上下文理解
- 除 Logo SVG 外，所有内容为文字（不把关键信息做在图片里）
- 不追求 JS 禁用下的降级体验；JS 禁用时 `<noscript>` 提示即可

### 4.6 禁忌清单（设计约束）

- ❌ 不使用渐变背景
- ❌ 不使用粒子特效
- ❌ 不使用卡通/emoji/可爱表情包
- ❌ 不使用3D渲染
- ❌ 不使用厚重阴影（box-shadow 仅允许 `0 1px 3px rgba(0,0,0,0.06)` 级别）
- ❌ 不使用AI生成感强烈的装饰元素
- ✅ 允许但克制：细线条装饰、猫狗剪影SVG、柔和留白

## 5. 宠物档案系统

### 5.1 数据模型

```javascript
// 单只宠物
const PetSchema = {
  id:        String,    // "pet_" + Date.now()，唯一标识
  name:      String,    // 宠物名字，必填，如"豆包"
  type:      String,    // "dog" | "cat"，必填
  breed:     String,    // 品种，选填，如"金毛"
  birthday:  String,    // 出生日期 "YYYY-MM"，选填
  weight:    Number,    // 体重(kg)，选填
  neutered:  Boolean,   // 是否绝育，默认false
  created:   String     // 创建日期 "YYYY-MM-DD"
};

// localStorage 存储结构
const StorageSchema = {
  pets:        PetSchema[],   // 所有宠物
  activePetId: String         // 当前选中宠物ID
};
```

### 5.2 存储策略

| 项目 | 决策 |
|------|------|
| 存储位置 | 浏览器 localStorage |
| 数据上传 | ❌ 绝不上传至任何服务器 |
| 跨设备同步 | ❌ 不支持（纯本地） |
| 数据丢失处理 | 用户清缓存后需重新创建，隐私提示条中说明 |
| 容量限制 | localStorage 5-10MB，单只宠物数据 <1KB，足够数百只 |

### 5.3 交互流程

```
用户首次进入首页
  └→ 左侧栏仅显示"+ 添加宠物"（桌面端≥1024px常驻；平板/手机端收起为顶部可折叠区域）
      └→ 点击 → 弹出添加宠物弹窗
          ├── 填写：名字(必填) + 猫/狗(必选) + 品种(可选) + 生日(可选) + 体重(可选) + 绝育(可选)
          ├── 点击"保存"
          └→ 宠物列表更新，自动选中新宠物
              └→ 点击任意工具 → 已填数据自动预填入计算器

已有宠物用户进入首页
  └→ 左侧栏显示所有宠物，默认选中上次活跃的
      └→ 点击其他宠物 → 切换激活
      └→ 点击"+ 添加宠物" → 同上流程
      └→ 宠物条目 hover 显示 ⋯ 图标 → 点击弹出编辑/删除菜单
          ├── 编辑：打开预填弹窗，修改后保存
          └── 删除：二次确认"确定要删除 [宠物名] 的档案吗？此操作不可撤销。"

进入工具页（宠物列表不显示）
  └→ 自动读取当前活跃宠物数据预填，保持工具页界面专注

用户点击工具（未创建宠物）
  └→ 弹出引导提示"创建宠物档案后，工具将自动预填年龄、体重等信息，省去每次手动输入。"
      [立即创建] [跳过，手动填写]

localStorage 不可用时
  └→ 首页顶部黄色提示条（可关闭）
  └→ 添加宠物按钮仍可点击但弹出说明
  └→ 工具页直接进入手动填写模式，不弹引导
```

### 5.4 工具页自动预填映射

| 工具 | 自动填入字段 | 用户只需补充 |
|------|------|------|
| 年龄换算 | type, birthday → 直接出结果 | 若为狗：体型选择 |
| 喂食量 | type, weight, neutered | BCS体况 + 活动水平 |
| 预产期 | type | 配种日期 |
| 驱虫周期 | type, birthday(→判定幼/成年) | 生活环境 + 上次驱虫日期 |
| 疫苗 | type, birthday → 直接出时间表 | 已接种勾选 + 生活环境 |

### 10.4 广告位技术设计（移动端优先）

设计原则：90% 用户手机访问 → 广告克制 → 只在自然阅读断点放置。

| 约束 | 移动端（<768px） | 桌面端（≥1024px） |
|------|------|------|
| 工具页广告位上限 | 2 个 | 2 个 |
| 首页广告位上限 | 1 个 | 1 个 |
| Blog 广告位上限 | 1 个 | 2 个 |
| 广告尺寸 | 320×100 / 300×250 | 728×90 / 300×250 |
| 禁止 | ❌ 插屏 ❌ 悬浮 ❌ 视频 | ❌ 插屏 ❌ 悬浮 |

参照《AdSense 审核通过指南》，分两阶段：

#### Phase 1：审核前 — CSS 占位符

不在页面放置任何广告代码。仅在布局上预留 CSS 占位 div。

移动端占位位置：
```
工具页：
  [计算器核心区]
  [结果展示]
  ── 📍 广告占位 320×100 ──   ← 第1个，自然断点
  [FAQ / 科普内容]
  ── 📍 广告占位 300×250 ──   ← 第2个（可选）
  [相关工具推荐]
  [页脚]

首页：
  [Hero]
  [工具卡片]
  ── 📍 广告占位 320×100 ──   ← 仅1个
```

占位样式：
```css
.ad-placeholder {
  border: 1px dashed var(--border);
  background: var(--bg-footer);
  display: flex;
  align-items: center;
  justify-content: center;
  color: var(--text-muted);
  font-size: 12px;
  min-height: 100px;
  margin: var(--space-md) 0;  /* 与内容区保持间距 */
}
/* 固定高度防止 CLS：实际广告加载后高度可能变化，预留 min-height 缓解 */
```

#### Phase 2：审核通过后

1. 搜索 `/* AD_PLACEHOLDER */` 注释，批量替换为 AdSense 代码
2. 启用 AdSense 自动广告 → **设为低密度** → 手动审核每个自动位置
3. 移动端严格 ≤2 个广告位，不得在输入区 100px 范围内放置
